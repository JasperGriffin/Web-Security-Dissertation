Broken authentication

Introduction

Broken authentication is marked as the second most vulnerable security risk and involves the poorly secured authentication where an attacker can compromise another user’s account. As a login page is the only system to elevate a user’s privilege on a web application, there are multiple strategies for an attacker to assume someone else’s identity.

Risks

Broken Authentication can result in the following factors:

- Automated attacks of a login system with leaked username and password pairs

- The eventual compromise of a user's account

Demonstration

If you haven't already, register as a user with an 4 letter or less username and password of the same amount with no capital letters or numbers. You may even want to login with this account for future purposes.

With this in mind, both of these values have a maximum entropy of roughly 37 bits. This is a measurement of username/password complexity or simply how hard are these values to guess.

Assuming this website has over 1,000 accounts where an attacker can guess at 1,000 times per second, then on average an account's username and password can get guessed in 0.1 seconds.

Solutions

From the perspective of a developer, the following solutions include:

Rate Limiting - Allow only a certain amount of login attempts on an account before locking it.

Two Factor Authentication - Continously log a user's location such that if their ip address doesn't match the original stored, send a verification code to the account's email to verify the user's legitimacy.

However, from the perspective of a user, you can also take the following initiatives:

Stronger password complexity - This includes having both a longer length and more characters. Password entropy is defined as the log of the character set to the power of the password length. Therefore, an 8 letter password with an upper-case and a lower-case character, a special symbol and a number increases its entropy to 52 bits. Or in terms of the scenario as before, will take roughly 250 years to guess.
